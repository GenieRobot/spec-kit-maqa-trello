You are setting up Trello integration for MAQA. This runs once.

## Prerequisites check

```bash
[ -n "$TRELLO_API_KEY" ] && echo "TRELLO_API_KEY: set" || echo "ERROR: TRELLO_API_KEY not set"
[ -n "$TRELLO_TOKEN" ]   && echo "TRELLO_TOKEN: set"   || echo "ERROR: TRELLO_TOKEN not set"
```

If either is missing, stop and tell the user:
1. Get API key at https://trello.com/app-key
2. Generate token from that page
3. Set `TRELLO_API_KEY` and `TRELLO_TOKEN` in their environment (`.env`, shell profile, or Claude Code settings)

## Step 1 — List accessible boards

```bash
BASE="https://api.trello.com/1"
AUTH="key=$TRELLO_API_KEY&token=$TRELLO_TOKEN"

curl -s "$BASE/members/me/boards?fields=id,name&$AUTH" | \
  python3 -c "
import json, sys
boards = json.load(sys.stdin)
for i, b in enumerate(boards):
    print(f\"{i+1}. {b['name']} ({b['id']})\")
"
```

Ask the user which board to use (by number or name). If the project name matches a board name, suggest it.

## Step 2 — Get lists for selected board

```bash
BOARD_ID="<selected board id>"

curl -s "$BASE/boards/$BOARD_ID/lists?fields=id,name&$AUTH" | \
  python3 -c "
import json, sys
lists = json.load(sys.stdin)
for l in lists:
    print(f\"{l['name']}: {l['id']}\")
"
```

Map lists to MAQA slots. Ask the user to confirm or correct if names don't match exactly:
- Backlog (optional)
- To Do → `todo_list_id`
- In Progress → `in_progress_list_id`
- In Review → `in_review_list_id`
- Done → `done_list_id`

If expected lists don't exist, offer to create them:

```bash
curl -s -X POST "$BASE/lists" \
  --data "name=To+Do&idBoard=$BOARD_ID&pos=bottom&$AUTH" | \
  python3 -c "import json,sys; print(json.load(sys.stdin)['id'])"
```

## Step 3 — Write config

```bash
mkdir -p maqa-trello

cat > maqa-trello/trello-config.yml << EOF
# MAQA Trello Configuration
# Generated by: /speckit.maqa-trello.setup — $(date -Iseconds)

board_id: "$BOARD_ID"
board_name: "$BOARD_NAME"

backlog_list_id: "$BACKLOG_LIST_ID"
todo_list_id: "$TODO_LIST_ID"
in_progress_list_id: "$IN_PROGRESS_LIST_ID"
in_review_list_id: "$IN_REVIEW_LIST_ID"
done_list_id: "$DONE_LIST_ID"

label_rules: {}
EOF
```

## Step 4 — Add to .gitignore (optional)

Ask the user if they want to gitignore the config (it contains no secrets, but board IDs are project-specific):

```bash
echo "maqa-trello/trello-config.yml" >> .gitignore
```

## Done

Report:
- Board: `$BOARD_NAME`
- Lists mapped: list each slot → list name → list ID
- Config written to: `maqa-trello/trello-config.yml`
- Next step: run `/speckit.maqa.coordinator` — Trello integration will activate automatically
